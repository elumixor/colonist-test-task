# Colonist — Checkered CTA A/B Variant

Take-home exercise for Colonist: a checkered 2×2 CTA layout to A/B test against
the current stacked homepage buttons.

**Live demo:** <https://elumixor.github.io/colonist-test-task/>

```
┌──────────┬──────────┐
│  orange  │          │
├──────────┼──────────┤
│          │  green   │
└──────────┴──────────┘
```

## What this is

Two buttons positioned diagonally (top-left + bottom-right), both locked to the
aspect ratio of the orange reference button, capped at 500 px wide, fully
responsive down to iPhone-SE widths. Built from the
[starter fiddle](https://jsfiddle.net/7wosc24n/) — see
[`docs/original-task-assignment.md`](docs/original-task-assignment.md) for the
full brief.

- **Primary (orange) — QUICK PLAY.** Pre-fetches `/api/room-list.json`,
  picks a real joinable room (humans inside, almost full, playable map &
  mode), and deep-links straight into it on click. Live count + seats-left
  shown on the button. Falls back to `#quickplay` if the proxy is
  unreachable or no rooms qualify.
- **Secondary (green) — LEADERBOARDS.** Pre-fetches
  `/api/leaderboards-tabs/`, which the upstream geo-personalises by IP.
  The button shows the active season + the viewer's continent/country
  ("S16 · EU"), then navigates to `/leaderboards` on click.

Both calls go through a 70-line Cloudflare Worker in [`proxy/`](proxy/)
that adds CORS headers (colonist.io's API doesn't ship them, so direct
browser fetches from github.io are blocked). The static fallback runs if
`PROXY_BASE` is empty in `submission/main.js`, so the demo never breaks
if the proxy is down or unset.

## Tech

Plain **HTML / SCSS / JavaScript** — no framework, no bundler, no TypeScript.

- `sass` compiles `submission/styles.scss` → `submission/styles.css`.
- `bun` runs the one build script and installs the one dev-dep.
- GitHub Actions compiles the SCSS and uploads `submission/` to Pages.

Zero runtime dependencies.

## Running locally

```bash
bun install
bun run dev          # one-shot build + `sass --watch` + browser-sync at :4173
                     # hot-reloads on any change in submission/ (CSS injected, no full reload)
```

Single-shot commands if you'd rather run the pieces separately:

```bash
bun run build        # compile submission/styles.scss → submission/styles.css
bun run watch        # sass --watch only
bun run serve        # browser-sync only (expects styles.css to exist)
```

## Design notes

- **Checkered grid.** A 2×2 CSS grid with the primary in cell (1,1) and the
  secondary in cell (2,2). Empty cells stay empty — no absolute positioning,
  no JS measurement.
- **Container queries, two scopes.** A named `slot` container on `.cta-slot`
  drives the grid (2-col ↔ 1-col at ~420 px). An unnamed container on each
  `.cta` drives typography — so title/subtitle size scale with the *button*,
  not the slot, which matters at tablet widths where the slot is wide but
  each cell is narrow.
- **Aspect ratio locked at 3.5.** Matches the orange reference and removes a
  class of layout bugs. `white-space: nowrap` on titles prevents two-line
  breaks like "JOIN A / LOBBY".
- **Flat vertical gradients.** No drop shadow, no 3-D ledge — matches the
  reference mock.
- **Typography.** Titles in Wicked Mouse (hot-linked from Colonist's CDN with
  open CORS). Subtitles in Vremena Grotesk Bold (drop a woff2 at
  `submission/fonts/` to activate; falls back to sans-serif otherwise).
- **API resilience.** 2.5 s `AbortController` per fetch, both calls run
  in parallel on load, all failures degrade silently to the static
  fallback (`#quickplay` / `/leaderboards`). The "live" pill only renders
  once data has actually landed, so it doubles as a visible health
  indicator — if it never appears, the proxy is down.
- **Proxy scope.** The Worker allowlists exactly four read-only GETs
  (`room-list`, `game-list`, `leaderboards-tabs`, `regions`). Any other
  path 404s. Per-route edge cache (15 s for room/game lists, 5 min for
  tabs, 1 h for regions) keeps upstream traffic minimal.
